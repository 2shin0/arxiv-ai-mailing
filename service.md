# arXiv AI Mailing 서비스 구조도

## 1. 개요

본 서비스는 매일 arXiv의 Computer Science (cs.AI) 카테고리에 새로 등록되는 논문을 자동으로 수집, 요약, 번역하여 이메일로 다이제스트를 발송하는 시스템입니다. 또한, 생성된 다이제스트는 GitHub 저장소에 마크다운 파일로 아카이빙됩니다.

## 2. 핵심 구성 요소

-   `main.py`: 전체 프로세스를 총괄하는 메인 스크립트입니다. 논문 크롤링부터 요약, 번역, 이메일 전송까지의 모든 단계를 조율합니다.
-   `crawler.py`: `arxiv` 라이브러리를 사용하여 arXiv.org에서 최신 AI 관련 논문 목록(제목, 저자, 초록, URL 등)을 가져옵니다.
-   `summarizer.py`: OpenAI의 GPT 모델을 사용하여 논문 초록을 영어로 요약하고, 이를 다시 한글로 번역합니다.
-   `email_sender.py`: 포맷팅된 HTML 다이제스트를 지정된 수신자에게 이메일로 발송합니다.
-   `exporter.py`: 처리된 논문 목록을 받아 마크다운 형식의 다이제스트 파일을 생성합니다.
-   `config.py`: API 키, 이메일 계정 정보, GitHub 저장소 URL 등 서비스 운영에 필요한 설정 값들을 관리합니다.
-   `deploy_digest.sh`: 생성된 마크다운 다이제스트 파일들을 GitHub 저장소에 자동으로 푸시하여 배포합니다.
-   `preview_email.py`: 실제 이메일 발송 전, 가상 데이터를 이용해 생성될 이메일의 모양을 HTML 파일로 미리 확인하는 유틸리티 스크립트입니다.

## 3. 실행 순서

1.  **논문 크롤링**: `main.py`가 `crawler.py`를 호출하여 arXiv에서 최신 AI 논문 목록을 가져옵니다.
2.  **요약 및 번역**: 각 논문에 대해 `summarizer.py`를 호출하여 초록을 영어로 요약하고, 다시 한글로 번역합니다. 이 과정의 진행 상황은 터미널에 실시간으로 출력됩니다.
3.  **논문 분류**: 논문의 제목과 초록을 분석하여 'LLM' 관련 논문을 별도로 분류합니다.
4.  **마크다운 다이제스트 생성**: `exporter.py`를 통해 LLM 관련 논문과 전체 논문에 대한 마크다운(`*.md`) 파일을 각각 생성하여 `digest/` 폴더에 저장합니다.
5.  **GitHub 배포**: `deploy_digest.sh` 스크립트를 실행하여 생성된 마크다운 파일들을 GitHub 저장소에 푸시합니다.
6.  **이메일 본문 생성**: `main.py`의 `make_digest` 함수가 이메일로 발송될 HTML 형식의 본문을 생성합니다.
7.  **이메일 발송**: `email_sender.py`를 통해 최종적으로 다이제스트 이메일을 발송하며 모든 과정이 마무리됩니다.
